Disk Scheduling Concepts

Disk Track

Numbered positions on the disk where data is stored (typically 0 to 199 in examples).

Disk Head

The read/write mechanism that moves between tracks to access data.

Seek Time

The time taken for the head to move from the current track to the requested track. Large head movement increases disk access time.

Request Queue

The list of upcoming track access requests pending service.

Algorithms Overview

FCFS

Processes requests in the order they arrive.

Head moves directly to next queued request.

SSTF

Chooses the request closest to the current head position.

Minimizes immediate seek time.

SCAN

Disk head moves like an elevator servicing requests in one direction to the end, then reverses.

Also known as the Elevator algorithm.

C-SCAN

Disk head moves in one direction to the end, then jumps back to the beginning without servicing.

Provides more uniform wait times.

LOOK

Similar to SCAN but stops at the last request instead of the disk edge.

More efficient than standard SCAN.

Why FCFS?

In FCFS (First-Come, First-Served) disk scheduling, requests are processed in the exact order they arrival in the queue. The disk head moves sequentially between requested tracks, and the total movement determines the disk access time.

Note: Because FCFS does not optimize movement, it can sometimes cause large head movement compared to other algorithms like SSTF or SCAN.

FCFS Algorithm Explanation

Steps

  1. Requests arrive in a queue.
  2. The disk head services them in arrival order.
  3. The head moves sequentially between tracks.
  4. Total head movement is calculated by summing absolute differences.

Example Input

Initial Head Position: 53

Requests: 98, 183, 37, 122, 14, 124, 65

Part 4 – Key Characteristics of FCFS

  • Simple and easy to implement: The algorithm requires minimal logic.
  • Processes requests in arrival order: Exactly like a first-in, first-out queue.
  • Fair scheduling method: Every request is serviced without starvation.
  • Can result in large head movement: Can wildly swing from 0 to 199 causing inefficiency (unlike SCAN or SSTF).

Conclusion

FCFS processes disk requests in arrival order but often causes large head movement. Algorithms such as SSTF, SCAN, C-SCAN, and LOOK optimize disk head movement by choosing requests based on proximity or direction. By interacting with the visualizations above, we can observe that optimizing scheduling drastically improves disk performance.
